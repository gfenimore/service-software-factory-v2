ViewForge V4 - Requirements Summary

Key Changes from V3 to V4:

‚úÖ Entity-First Workflow:

  • Start with entity selection from BUSM.mmd
  • Select 3 primary + supporting entities
  • Entities drive everything else (fields, relationships, modals)

‚úÖ Master View Pattern Support:

  • 3-column layout as primary pattern
  • Contextual modal configuration
  • Data flow: Column 1 ‚Üí Column 2 ‚Üí Column 3

‚úÖ SME-Driven Configuration:

  • Subject Matter Expert picks entities based on business knowledge
  • No document scanning or field-first configuration
  • Configuration exports directly to Stage 1 pipeline

‚úÖ Integration with Factory:

  • Reads from actual BUSM.mmd file
  • Exports configuration that feeds Stage 1
  • Bridges Discovery Canvas findings to pipeline execution

üéØ Success Criteria:

  • SME can configure Master View in under 30 minutes
  • Configuration directly drives Concept Line ‚Üí Prototype Line ‚Üí Production
  • No technical knowledge required for entity selection
  • Supports contextual modals for "absolute least clicks" requirement
