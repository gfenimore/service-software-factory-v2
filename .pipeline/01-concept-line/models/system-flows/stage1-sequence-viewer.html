üéØ Stage 1 Overview - Service Software Factory Template-Based

Purpose: Stage 1 captures scope via UI form, selects entities directly from BUSM, auto-generates components for 3-column layout, and creates business rules through 3-layer inheritance (Base ‚Üí Service ‚Üí Client), with template save/load for rapid service delivery.

  • Input: UI form (Service + Client + Module) + Direct entity selection from BUSM.mmd
  • Processing: Auto-generate 3-column Master View + 3-layer business rules + template creation
  • Output: scope-definition.json, selected-entities.json, component-specifications.json, business-rules.json, template-config.json
  • Result: Service Software Factory template created - 90% standardization + 10% customization
